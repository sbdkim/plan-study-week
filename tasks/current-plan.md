# Current Plan

## Goal

Create a Northline-branded study planner that deploys cleanly to GitHub Pages with vanilla HTML, CSS, and JavaScript, while leaving a clean path for optional Firebase free-tier auth and sync later.

## Implementation Steps

1. Scaffold the project structure and baseline docs.
2. Build a polished single-page app shell with a responsive dashboard.
3. Add local storage state for subjects, tasks, and study sessions.
4. Seed demo content so the first load feels complete.
5. Review responsiveness, interaction states, and GitHub Pages readiness.
6. Apply the shared Northline visual system, naming, and favicon so the planner matches the suite foundations.

## Verification

- Manual smoke check for add, toggle, delete, and persisted reload flows
- Review layout at mobile and desktop breakpoints
- Confirm the app runs as a static site with no build step
- Confirm the shared Northline title, fonts, favicon, and token stylesheet load correctly
